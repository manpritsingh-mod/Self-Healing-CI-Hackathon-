## Description

> Every day, software teams lose hours waiting for someone to investigate why a CI/CD build broke. Our project eliminates that wait entirely.
>
> **Self-Healing CI/CD Engine** is an intelligent system that sits alongside Jenkins and automatically repairs broken builds — without any human intervention. The moment a build fails, the engine receives a webhook, fetches the failure logs, and hands them off to a pipeline of specialized AI agents. Each agent has one job: one parses logs, one identifies the root cause, one writes the code fix, and one validates whether the fix is actually correct. These last two agents work in a loop — challenging each other back and forth until they reach at least 90% confidence that the fix will work.
>
> Once validated, the fix is committed to a new Git branch, a Pull Request is opened automatically on GitHub, and Jenkins is re-triggered to verify the repair — all without a developer touching anything. A Slack notification is sent with full details and interactive Accept/Reject buttons so the team stays informed and in control.
>
> The engine also remembers every fix it produces. Using a vector database, it matches new failures against past solutions — resolving recurring issues instantly without spending any AI tokens. This makes the system smarter and cheaper over time.
>
> Built with Python (FastAPI), ChromaDB, Docker, and supports multiple LLM providers including Gemini, OpenAI, and local Ollama.
